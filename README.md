# 💪 Kropp Fitness — Лендинг начального уровня

Адаптивный одностраничный сайт (лендинг) для фитнес-клуба. Проект создан для портфолио с целью демонстрации навыков чистой верстки, работы с препроцессором SCSS и модульной архитектуры стилей.

---

## 🛠️ Стек технологий и инструменты
* **HTML5** — семантическая разметка.
* **SCSS (Sass)** — препроцессор для стилизации (использование модулей `@use`/`@forward`, переменных, миксинов и вложенности).
* **BEM (БЭМ)** — методология именования классов для поддержки и масштабируемости кода.
* **WebStorm** — интегрированная среда разработки (IDE).
* **Sass File Watcher** — для автоматической локальной компиляции SCSS в CSS.
* **Git & GitHub** — контроль версий и хостинг проекта.

---

## 📂 Архитектура SCSS стилей
В проекте реализована модульная структура стилей по принципу «от общего к частному»:
* `styles/` — общая папка стилей.
    * `0-globalstyles/` — глобальные настройки: сброс стилей (normalize), шрифты, CSS-переменные. Папка собирается через `_index.scss` и пробрасывается дальше с помощью `@forward`.
    * `header/` — стили для шапки сайта. итд 
    * `styles.scss` — главный файл-диспетчер, собирающий все модули воедино.

 Этап 1: Подготовка проекта, настройка SCSS и базовые стили
</summary>

На данном этапе выполнена полная подготовка проекта к работе, настроена удобная структура папок и заложен фундамент для будущей верстки.

* **Настройка автоматической сборки стилей**:
    * Настроена автоматическая компиляция SCSS в обычный CSS прямо во время сохранения файлов в WebStorm.
    * Создан главный файл-диспетчер `styles.scss`, собирающий все стили воедино.

* **Организация структуры папок**:
    * Все глобальные стили вынесены в отдельную папку `0-globalstyles`.
    * Использованы современные директивы `@use` и `@forward` для аккуратного подключения файлов друг к другу без конфликтов в коде.

* **Подключение сброса стилей (`_normalize.scss`)**:
    * Интегрирован современный сброс стилей, устраняющий стандартные различия в отображении элементов между браузерами.
    * Обеспечена адаптивность картинок по умолчанию и настроена плавная прокрутка страницы.

* **Подключение шрифтов и настройка текста (`_fonts.scss`, `globals.scss`)**:
    * Локально подключены шрифты **Heebo** (для обычного текста) и **Yantramanav** (для заголовков).
    * Заданы базовый размер текста, комфортный межстрочный интервал (line-height) и автоматический перевод всех заголовков в верхний регистр.

* **Создание глобальных переменных (`_variables.scss`, `globals.scss`)**:
    * В `:root` вынесены все ключевые параметры: цвета проекта (черный, белый, серые оттенки), радиусы скругления углов, рамки и время анимации для удобного изменения темы в одном месте.
    * Создан класс ограничивающего контейнера `.container` (максимальная ширина контента 1300px с боковыми отступами по 15px), предотвращающий прилипание элементов к краям экрана.

* **Порядок в репозитории**:
    * Настроен файл `.gitignore` для исключения системных файлов (папка `node_modules`, настройки WebStorm и отчеты Repomix) из отслеживания Git.

Этап 2: Разметка и стилизация адаптивной шапки (Header) и интеграция JS
</summary>

На данном этапе полностью завершена работа над шапкой сайта (Header): созданы разметка, стили, анимации, адаптивный оверлей и подключен интерактивный JavaScript для мобильной версии.

* **Семантическая разметка (HTML5 & BEM)**:
    * Написана разметка шапки с использованием логотипа, навигации по сайту и целевой кнопки «Book now».
    * Мобильное меню заверстано с использованием современного тега `<dialog>` для лучшей доступности.
    * Интегрирована кастомная SVG-кнопка бургера с подготовленными путями для анимации.

* **Интерактивный JavaScript (ES-Modules)**:
    * Написан масштабируемый класс `OverlayMenu` в виде отдельного модуля.
    * Реализовано переключение состояния кнопки бургера (`is-active`), открытие окна `<dialog>` и блокировка скролла всей страницы при открытом меню путем добавления класса `is-lock` на тег `html`.
    * Скрипт подключен как модуль (`type="module"`) в `<head>` документа.

* **Анимации и стилизация шапки (`_header.scss`, `_burger-button.scss`, `_button.scss`)**:
    * Шапка зафиксирована при прокрутке экрана (`position: sticky`, `top: 0`).
    * С помощью современного CSS (`animation-timeline: scroll()`) реализовано плавное уменьшение высоты шапки при прокрутке страницы вниз.
    * Реализовано плавное появление полноэкранного мобильного меню справа налево через свойства `opacity`, `translate` и современное правило `@starting-style` с поддержкой `transition-behavior: allow-discrete`.
    * Настроена красивая CSS-анимация превращения бургера в крестик при клике за счет смещения параметров `stroke-dasharray` и `stroke-dashoffset` у SVG-путей.
    * Вынесена стилизация кнопок в отдельные переиспользуемые модули.

<summary>
Этап 3: Верстка первого экрана (Banner), оптимизация утилит и интерактивная пагинация
</summary>

На данном этапе завершена разработка первого экрана (Banner): создана семантическая разметка контента, настроена гибкая пагинация и оптимизирована система глобальных SCSS-миксинов.

* **Разметка и стилизация баннера (`_banner.scss`, `index.html`)**:
    * Секция `.banner` спозиционирована на всю высоту экрана за вычетом высоты шапки с использованием динамических единиц `dvh`: `min-height: calc(100dvh - 70px)`.
    * Реализовано вертикальное выравнивание контента баннера к нижней части экрана с использованием флексбокс-структуры.
    * Интегрирован сверхкрупный заголовок с адаптивным расчетом размера шрифта: `font-size: min(360px, 18.75vw)`.
    * Добавлен декоративный элемент стрелки-указателя, адаптивно масштабируемый под разные разрешения экрана.

* **Адаптивная пагинация и эффекты наведения**:
    * Сверстана пагинация слайдера баннера. Интерактивные точки реализованы через псевдоэлементы `::after` с плавной сменой состояний.
    * Настроена правильная логика взаимодействия: наведение происходит на кнопку-родителя, после чего перекрашивается внутренний псевдоэлемент `::after`. Эффект применен только к неактивным точкам (`:not(.is-current)`).

* **Глобальные миксины доступности (`_maxins.scss`, `_utils.scss`)**:
    * Создан файл `_maxins.scss` и внедрен профессиональный миксин `@mixin visually-hidden` для скрытия элементов разметки с экрана без потери их доступности для программ экранного доступа (скринридеров).
    * Внедрен служебный класс `.visually-hidden` для скрытия вспомогательных заголовков секций (`h1`, `h2`).
    * Файлы утилит успешно интегрированы в общую сборку через центральный `0-globalstyles/_index.scss`.

<summary>
 Этап 4: Разработка секции Motivation (шахматная сетка, декоративные подложки и оптимизация производительности)
</summary>

На данном этапе реализована контентная секция Motivation, настроена сложная адаптивная сетка карточек, внедрены кастомные декоративные элементы и устранена проблема скачков интерфейса при прокрутке.

* **Шахматное позиционирование элементов (`_motivation.scss`, `index.html`)**:
    * Реализована сетка карточек мотивации на основе флексбокс-модели.
    * Применен принцип контекстного позиционирования: зеркальный разворот карточек реализован через обращение к порядковому номеру родительского элемента (`.motivation__item:nth-child(even) .motivation__card { flex-direction: row-reverse }`). Это позволило сохранить независимость и чистоту дочернего класса `.motivation__card`.

* **Декоративные элементы и доступность (`_backdrop-title.scss`, `_title-medium.scss`)**:
    * Внедрен переиспользуемый класс `.backdrop-title` для создания крупных фоновых текстовых подложек позади заголовков.
    * Текстовая подложка реализована через псевдоэлемент `::after` и CSS-функцию `content: attr(data-title)`. Это позволило не перегружать HTML-код лишними тегами, сохранив семантическую чистоту разметки и доступность для скринридеров (декоративный текст игнорируется программами чтения экрана).
    * Псевдоэлементу `::after` задано абсолютное позиционирование, что автоматически перевело его тип отображения в блочный без необходимости явно прописывать свойство `display: block`.
    * Элемент-маркер (плюс) привязан к размеру шрифта заголовка с использованием относительных единиц `em` для сохранения идеальных пропорций при масштабировании текста.

* **Разработка прозрачной интерактивной кнопки**:
    * Стилизована прозрачная кнопка `.button--transparent` со встроенной стрелкой в псевдоэлементе `::after`.
    * Реализована правильная иерархия наведения: событие hover обрабатывается на самой кнопке, плавно перекрашивая внутренний псевдоэлемент (`&::after { filter: brightness(0.75) }`).

* **Оптимизация скролла (Устранение скачков)**:
    * Найдена и устранена проблема резких скачков страницы при скролле вниз с баннера. Отрицательные вертикальные отступы картинок карточек (`margin-block: -60px`) заменены на безопасный сдвиг через `transform: translateY()`, что стабилизировало расчет высоты страницы браузером.

На данном этапе разработана контентная секция видов тренировок (Training Types): создана гибкая Grid-сетка с трехступенчатой адаптивностью и оптимизировано управление фоновыми ресурсами при помощи локальных CSS-переменных.

* **Адаптивная CSS Grid сетка (`_training-types.scss`, `index.html`)**:
    * Список видов тренировок реализован на базе технологии CSS Grid с автоматическим распределением свободного пространства.

* **Оптимизация фоновых изображений через локальные переменные**:
    * Декоративные угловые стрелки интегрированы как фоновое изображение элементов списка (`__item`).
    * Для управления пространством под стрелку внедрена локальная CSS-переменная `--trainingTypesItemBgIconSize` со значением по умолчанию `80px`.
    * При адаптации под мобильные экраны изменено только значение этой переменной (`--trainingTypesItemBgIconSize: 60px`), что позволило избежать повторного дублирования длинного пути `background: url(...)` в медиа-запросах.
